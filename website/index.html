Problem & approach

The problem

Large Python codebases accumulate v6 patterns: middleware kwargs, provider names, filter kwargs, and sign-and-send middleware. Manual upgrades are slow and error-prone.

Our approach

Deterministic rules with 9 JSSG fixtures, optional AI follow-up in the workflow, and documented limits (e.g. conservative handling of POA factory-style code).

What the codemod handles

  • middlewares= → middleware=
  • WebsocketProviderV2 → WebSocketProvider
  • RPC-style kwargs (fromBlock, toBlock, blockHash) → snake_case, with guards for bare dict(...)
  • Selected middleware-onion bare identifiers → class instances + import cleanup
  • construct_sign_and_send_raw_middleware → SignAndSendRawMiddlewareBuilder.build(...) where applicable

Try it locally

From the repository root:

cd packages/web3py-v6-v7
npm install
npm test
npx codemod workflow validate -w workflow.yaml

Apply to a project (use a branch first):

npx codemod workflow run -w workflow.yaml -t /path/to/repo \
  --allow-dirty --no-interactive --allow-fs
